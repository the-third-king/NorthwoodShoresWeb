Breakdown of Expectations

  1. We have made our website for a campground in Rice Lack called Northwood Shores Campground
  2. We have a Northwood Shores Campground header on every page with the slogan
  3. We have 10 different pages in the home, amenities, Rates, site map, seasonal sites, the lake, location , area attractions, photo gallery, and the contact pages
  4. We have a navigation bar on the left of every page
  5. We have css cards that shrink down the number of columns in that page. Also our nav bar will shrink down to a hamburger menu when it is small enough.
  6. We mainly use css cards and the same nav bar and header on every page so it looks coherent.
  7. All images have an alt category.
  8. Every page has the same campfire favicon since it is a campground
  9. We have 2 folders for css and images and leave the html out in the parent folder for easy access to get to.
  10. We only used css and html. There is a javascript file in here but that method is commented out for the time being.
  11. Every page passes the html and css validator.
  12. We have a github at Northwood Shores Github.
  13. All files are lowercase using alphanumeric letters only.
  14. All content is aligned in there own way to make pages look cohesive.
  15. We have 2 different font faces for the pages and for the header bar.
  16. we have links to external sites in the area attractions page and also a link to facebook in the footer.
  17. The source code is readable using understandable names.
  18. The nav bar is the same on every page and even when shrunk down it is consistent.
  19. Every webpage has a header and footer section with the navbar.
  20. We use mainly flexboxes in all of our pages.

Project Part 2 Expectations

  1. All javascript files were placed in a folder called js
  2. We have four web pages implementing forms in the contact us page, available sites page, admin sign in page, and admin site changing page
  3. We have a photo carousel on the homepage as our javascript animation
  4. Our website uses MySQL using XAMPP do collect all of our data
  5. We implemented an admin sign in page with login (Username: admin, password: admin) for the admin to change campsite availability
  6. We addressed the feedback of our website colors being off, our text being too small, our navigation bar looking weird, and styling our webpages a little better
